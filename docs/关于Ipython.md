# IPython 完整介绍
## 一、简单一句话定义
IPython 是**增强版 Python 交互式解释器**，在原生 `python` 终端基础上增加大量实用功能，后来衍生出 Jupyter Notebook/Jupyter Lab，我们平时写代码用的网页笔记本内核底层就是 IPython。

原生 Python 终端很简陋，IPython 解决了它所有痛点。

## 二、两种使用形态
### 1. 命令行终端版（ipython）
在终端输入 `ipython` 直接启动，界面示例：
```
In [1]: print("hello ipython")
hello ipython

In [2]: a = 10

In [3]: a * 2
Out[3]: 20
```
特征：
- 输入行 `In [序号]:`，输出行 `Out[序号]:`
- 彩色语法高亮、自动缩进、语法报错彩色提示
- 支持 Tab 自动补全函数、变量、文件、模块

### 2. Jupyter 内核（最常用）
Jupyter Notebook / Lab 背后执行代码的引擎就是 IPython，网页里分单元格运行代码、实时绘图、写 Markdown，本质都是 IPython 提供交互能力。

## 三、核心特色功能（对比原生 python）
### 1. 输入输出历史管理
- `_`：上一条输出结果
- `__`：上两条输出
- `_N`：第 N 号 Out 输出，例如 `_3` 取 `Out[3]`
- `%history`：查看全部执行过的代码

```
In [1]: 1+2
Out[1]: 3

In [2]: _ * 5
Out[2]: 15
```

### 2. 魔法命令（% 行魔法 / %% 单元格魔法）
IPython 独有的命令，不是 Python 语法，以 `%` 开头：
- `%ls` / `%pwd`：查看当前目录、路径（系统命令）
- `%cd`：切换文件夹
- `%time`：测算单行代码运行时间
- `%%timeit`：循环多次精准测速
- `%matplotlib inline`：让绘图直接嵌入输出
- `%run test.py`：运行外部 py 文件
- `%load test.py`：把文件代码加载到单元格

### 3. 系统 Shell 交互
以 `!` 开头直接执行系统命令：
```
In [1]: !ls
In [2]: !pip list
```
还能把命令结果赋值给变量：
```
files = !ls
```

### 4. 强大帮助与文档
- `函数?`：查看函数文档字符串
- `函数??`：查看完整源码
- `help(xxx)` 也增强排版，可读性远好于原生 python

```
In [1]: print?
In [2]: import numpy; numpy.array??
```

### 5. 自动补全与自省
输入一半按 Tab：自动补全变量、函数、模块、文件路径；
输入 `.` 后 Tab 列出对象所有属性、方法。

### 6. 便捷调试、重载模块
- `%debug`：报错后自动打开调试器，查看变量
- `%reload_ext autoreload` 自动重载修改过的模块，不用重启解释器

### 7. 美观绘图、多媒体输出
原生 Python 终端只能打印文字；
IPython 支持直接输出图片、表格、HTML、音频、视频，Jupyter 里靠它实现可视化。

## 四、安装与启动
### 安装
```bash
pip install ipython
# 同时装jupyter（推荐）
pip install ipython jupyter
```

### 启动终端版
```bash
ipython
```

### 启动网页笔记本（基于IPython内核）
```bash
jupyter notebook
# 或新版jupyter lab
jupyter lab
```

## 五、IPython 和原生 python 的直观对比
1. **原生 python**
   - 提示符 `>>>`，无行号区分输入输出
   - 无语法高亮、无补全、无魔法命令
   - 不能直接运行系统命令，绘图弹出单独窗口
   - 历史记录简陋，无法快速调取过往输出

2. **IPython**
   - `In [x]/Out[x]` 带编号交互
   - 彩色高亮、Tab补全、丰富魔法工具
   - 内嵌 shell、计时、调试、文件操作
   - 完美支持数据科学可视化，是数据分析标配

## 六、适用场景
1. 快速测试小段代码、验证函数逻辑
2. 数据分析、机器学习实时调试（Jupyter）
3. 教学、笔记式代码记录（代码+文字+图表一体）
4. 临时运行脚本、快速查看文件与环境信息

简单总结：IPython 就是**专为交互式开发、数据分析设计的增强 Python 环境**，现在几乎所有数据从业者、学生写代码都会使用它。

# 一、先解释你报错 `_1 NameError` 的原因
`_N` 指代的是 `Out[N]`，**只有那一行代码有输出结果，才会生成 `_N`**；
如果单元格只是执行变量赋值、函数定义、循环、无返回值语句，没有 `Out[x]`，`_x` 这个变量根本不存在，调用就会报 NameError。

举例子：
```python
# In[1]: 无输出，Out[1]不存在
a = 10
# 此时 _1 未定义，执行 _1 直接报错

# In[2]: 有输出，Out[2]存在
a + 5
# 此时 _2 = 15，可以正常调用
```
而且你是网页 Jupyter（远程内核，不是本地终端ipython），规则完全一样，这个报错和网络远程运行无关。

# 二、查看历史代码的多种可靠方法（Jupyter/IPython 通用，远程网页也能用）
## 方法1：魔法命令 %history（最推荐，完整看所有代码）
直接运行：
```python
%history
```
效果：打印全部单元格执行过的源码，不管有没有输出，全部记录。

### 常用参数
```python
# 只看第5到10号单元格代码
%history 5-10

# 只看最近3条
%history -n 3

# 把历史代码保存到 py 文件
%history -f code_backup.py
```

## 方法2：%recall 调取指定单元格代码
```python
# 把 In[12] 的代码重新填充到当前单元格，直接回车就能再运行
%recall 12
```
适合想复用某一段旧代码，不用手动复制。

## 方法3：读取输入历史变量 In
IPython 内置全局列表 `In`，存了**所有单元格输入代码**，不受有无输出影响，完美解决 `_N` 失效问题：
```python
# 查看第12单元格原始代码
print(In[12])

# 打印全部历史代码
for idx, code in enumerate(In):
    if idx > 0:
        print(f"In[{idx}]:\n{code}\n")
```
`In` 永远存在，只要单元格执行过就会存入，不会报 NameError。

## 方法4：网页界面自带操作（不用写代码）
1. **向上滚动页面**：直接回看旧单元格，自带代码+运行结果；
2. 菜单栏 `Kernel` → `Reconnect` 断连重连不会清空历史（只要没重启内核）；
3. 顶部 `Edit` → `Find and Replace` 全局检索历史代码关键字；
4. 导出全部历史：`File → Download as → Python (.py)`，会导出从头到尾所有单元格代码。

## 方法5：保存全部历史到文件永久留存
```python
# 导出所有执行过的代码到脚本
%history -f all_history.py
```
即使重启内核，文件里也保留全部代码记录。

## 方法6：%save 单独保存某几个单元格代码
```python
# 把 In[3]、In[5]、In[7] 的代码存到 temp.py
%save temp.py 3 5 7
```

# 三、区分两个容易混淆的内置变量（重点）
1. `In[数字]`：**全部输入代码**，一定存在，用来回看代码；
2. `_数字` / `Out[数字]`：**仅单元格输出结果**，无输出则不存在，只能拿返回值，不能看代码；

你刚才想拿代码却用了 `_1`，相当于拿输出值，自然报错；想看代码统一用 `In[1]` 或者 `%history`。

# 四、远程网页 Jupyter 额外小提示
网络通信模式不会限制上面所有魔法命令和 `In` 变量，逻辑和本地终端 IPython 完全一致；
只有两种情况历史会清空：
1. 点击 `Kernel → Restart Kernel` 重启内核；
2. 关闭页面、服务被停止，内核进程销毁。
单纯刷新网页、断网重连，代码历史都会保留。
